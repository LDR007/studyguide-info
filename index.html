SQL Language

SQL: Concetti Base

SQL è il linguaggio standard (inizialmente Structured Query Language del System R di IBM, anni '70) per i DBMS relazionali. Comprende:

  • DDL (Data Definition Language): Definizione dello schema (tabelle, domini, viste, vincoli).
  • DQL (Data Query Language): Interrogazione dei dati (SELECT).
  • DML (Data Manipulation Language): Modifica dei dati (INSERT, UPDATE, DELETE).
  • DCL (Data Control Language): Controllo degli accessi e permessi.

I Domini Elementari (Tipi di Dato)

  • Caratteri: char(n) (fisso), varchar(n) (variabile). Possibile specificare il character set (es. Greek).
  • Numerici Esatti: numeric(P,S), decimal(P,S), integer, smallint, bigint (SQL-3). P = precisione (cifre totali), S = scala (cifre decimali).
  • Numerici Approssimati: float(P), real, double precision (mantissa ed esponente).
  • Temporali (SQL-2): date (anno, mese, giorno), time (anche with time zone), timestamp, interval (durate, es. year to month).
  • SQL-3 (LOB & Booleani): boolean, blob (Binary Large Object), clob (Character Large Object).

DDL: Definizione dello Schema e Cataloghi

sqlCREATE SCHEMA [Nome] [ AUTHORIZATION Utente ];
CREATE DOMAIN Nome AS Tipo [ DEFAULT Valore ] [ CHECK (Condizione) ];
CREATE TABLE Dipartimento (
  Nome varchar(20) PRIMARY KEY,
  Indirizzo varchar(50) NOT NULL,
  Città varchar(20) DEFAULT 'Milano'
);

Cataloghi Relazionali (Riflessività): I DBMS gestiscono i propri metadati (dizionario dei dati) tramite tabelle relazionali interne. I comandi DDL manipolano queste tabelle in modo sicuro e portabile.

Vincoli di Integrità e Asserzioni

Vincoli Intrarelazionali (Singola tabella)

  • NOT NULL: Impedisce valori nulli. Se assente e non c'è default, vale NULL.
  • UNIQUE: Definisce una superchiave (valori tutti diversi, esclusi i NULL multipli).
  • PRIMARY KEY: Identificatore unico. Implica NOT NULL e UNIQUE.
  • CHECK (Condizione): Vincolo generico complesso a livello di riga o tabella.

Integrità Referenziale (Foreign Key)

sqlFOREIGN KEY (ListaAttr) REFERENCES TabellaMaster(ListaAttr)
  [ON DELETE Politica] [ON UPDATE Politica]

Politiche di reazione (su tabella esterna): CASCADE (propaga modifica/cancellazione), SET NULL, SET DEFAULT, NO ACTION (rifiuta - default).

Asserzioni e Politiche di Controllo

Le Asserzioni (CREATE ASSERTION) sono vincoli globali di schema slegati da singole tabelle (utili per cardinalità minime e vincoli multitabella).

  • Controllo Immediato: Verificato dopo ogni singola modifica (es. vincoli predefiniti).
  • Controllo Differito (DEFERRED): Verificato solo alla fine della transazione al COMMIT (essenziale per vincoli incrociati).

Modifica Schemi: ALTER e DROP

sqlALTER TABLE Dipartimento ADD COLUMN NroUff numeric(4);
ALTER DOMAIN Nome SET DEFAULT 'Anonimo';

DROP TABLE NomeTabella [ RESTRICT | CASCADE ];
  • RESTRICT: Fallisce se l'oggetto non è vuoto o è referenziato (default).
  • CASCADE: Rimuove l'oggetto e tutto ciò che dipende da esso.

DQL: Interrogazione dei Dati (SELECT)

Il DQL (Data Query Language) si occupa di estrarre e leggere i dati. In SQL, le interrogazioni sono dichiarative: indichi cosa vuoi ottenere e non come (logica basata sul calcolo relazionale).

Struttura di base (Clausole principali)

  • SELECT: Specifica le colonne o espressioni. Usa * per estrarre tutti gli attributi o DISTINCT per rimuovere i duplicati.
  • FROM: Elenca le tabelle su cui operare. È possibile usare degli Alias per rinominare temporaneamente le tabelle, utili per query complesse o auto-join.
  • WHERE: Filtra le righe tramite espressioni booleane. SQL usa logica a tre valori (True, False, Unknown). Supporta il confronto con i NULL tramite IS NULL o IS NOT NULL.

Join (Interni ed Esterni)

  • INNER JOIN: (o Theta-join) Restituisce solo le righe che trovano corrispondenza in entrambe le tabelle.
  • OUTER JOIN (LEFT, RIGHT, FULL): Mantiene nel risultato anche le righe senza corrispondenza nell'altra tabella, riempiendo i campi mancanti con NULL.
  • NATURAL JOIN: Join implicito su colonne con lo stesso nome. 
sqlSELECT I.Nome, D.Città 
FROM Impiegato I LEFT JOIN Dipartimento D 
ON I.Dipart = D.Nome;

DQL: Ordinamento, Aggregati e Raggruppamento

I risultati possono essere ordinati tramite la clausola ORDER BY, in modo ASC (ascendente, default) o DESC (discendente).

Funzioni di Aggregazione

Operatori per calcolare valori su insiemi di righe: COUNT(*), COUNT(DISTINCT Attr), SUM, AVG, MIN, MAX.

Raggruppamento (GROUP BY e HAVING)

  • GROUP BY: Divide le righe in sottoinsiemi basati su attributi comuni per applicare gli aggregati a ciascun gruppo.
  • HAVING: Filtra i gruppi creati dal GROUP BY in base ai risultati delle funzioni aggregate (agisce come una WHERE sui gruppi). 
sqlSELECT Dipart, AVG(Stipendio) FROM Impiegato
GROUP BY Dipart HAVING AVG(Stipendio) > 2500;

Operatori Insiemistici

Combinano i risultati di più query: UNION, INTERSECT, EXCEPT (o MINUS). Di default eliminano i duplicati; usa la parola chiave ALL per mantenerli.

DQL: Interrogazioni Nidificate

Permettono di confrontare attributi con i risultati di una subquery tramite operatori come ANY, ALL, IN, NOT IN ed EXISTS (verifica se il risultato non è vuoto). 

sql-- Impiegati che guadagnano più della media
SELECT Nome, Stipendio FROM Impiegato 
WHERE Stipendio > (SELECT AVG(Stipendio) FROM Impiegato);

-- Query correlata con EXISTS
SELECT * FROM Persona P 
WHERE EXISTS (
  SELECT * FROM Persona P1 
  WHERE P1.Nome = P.Nome AND P1.CodFiscale <> P.CodFiscale
);

DML: Modifica dei Dati

  • INSERT INTO: Singole righe (con VALUES) o interi set di righe (con subquery SELECT).
  • DELETE FROM: Rimuove le righe che soddisfano la condizione. Se si omette la WHERE, svuota la tabella.
  • UPDATE: Aggiorna valori degli attributi. Nota: in SQL l'aggiornamento è set-oriented; ordini multipli di update potrebbero accavallarsi e produrre risultati inattesi se le condizioni si sovrappongono. 
sqlINSERT INTO ProdottiMilanesi (SELECT Codice, Descrizione FROM Prodotto WHERE Luogo = 'Milano');

UPDATE Impiegato SET Stipendio = Stipendio * 1.1 WHERE Dipart = 'Amministrazione';

DELETE FROM Dipartimento WHERE Nome NOT IN (SELECT Dipart FROM Impiegato);

DCL: Controllo Accesso e Transazioni

Il DCL (Data Control Language) gestisce la protezione dei dati stabilendo chi accede alle risorse (tabelle, viste, attributi) e quali operazioni può eseguire.

I Privilegi

Il controllo si basa sui privilegi. Il proprietario della risorsa ha tutti i privilegi. L'utente _system (amministratore) ha privilegi totali.

  • SELECT, INSERT, UPDATE, DELETE: Lettura e modifica.
  • REFERENCES: Permesso di riferirsi a risorse esterne (es. per Foreign Keys).
  • USAGE: Uso di domini.

Comandi GRANT e REVOKE

  • GRANT: Concede privilegi. Opzione WITH GRANT OPTION per permettere la propagazione ad altri.
  • REVOKE: Sottrae privilegi. Opzioni RESTRICT (blocca se ci sono dipendenze) o CASCADE (forza la rimozione a catena). 
sqlGRANT SELECT, UPDATE ON Dipartimento TO Stefano WITH GRANT OPTION;
REVOKE ALL PRIVILEGES ON Impiegato FROM Paolo CASCADE;

Modello a Ruoli (RBAC in SQL-3)

Un Ruolo (CREATE ROLE) è un contenitore di privilegi. Gli utenti li attivano con SET ROLE. Garantisce il principio del "minimo privilegio": l'utente dispone solo delle autorizzazioni necessarie al compito attuale. 

sqlCREATE ROLE Manager;
GRANT SELECT, UPDATE ON Impiegato TO Manager;
GRANT Manager TO Stefano;
SET ROLE Manager; -- Stefano attiva il ruolo

Transazioni e ACID

Una transazione è un'unità elementare di lavoro atomica (START TRANSACTION, COMMIT, ROLLBACK).

  • Atomicità: "Tutto o niente" (gestita via Undo/Redo).
  • Consistenza: La transazione porta il DB da uno stato coerente a un altro (rispetta i vincoli).
  • Isolamento: Esecuzione indipendente dalle altre transazioni concorrenti.
  • Persistenza: Gli effetti del COMMIT non vanno mai persi.

Viste

Tabelle virtuali il cui contenuto è calcolato dinamicamente tramite una SELECT. 

sqlCREATE VIEW NomeVista AS SELECT ... WITH CHECK OPTION;
  • Aggiornabilità: Consentita di solito solo se definita su una singola tabella base.
  • WITH CHECK OPTION: Impedisce modifiche alla vista che farebbero uscire la riga dalle condizioni della vista stessa. Opzioni: LOCAL (controlla solo l'ultimo livello) o CASCADED (default, controlla tutti i livelli nidificati).

Ottimizzazione: Indici

Strutture fisiche (B-Tree, Hash) per velocizzare le query senza scansione completa. 

sqlCREATE [ UNIQUE ] INDEX IdxCognome ON Impiegato (Cognome);

Query Avanzate (SQL Moderno)

CTE (WITH): Rendono le query leggibili e supportano la ricorsione. Supportate da PostgreSQL, MariaDB, SQL Server.

Window Functions (OVER): Calcoli su set di righe senza raggruppamento (es. RANK(), AVG() OVER).

SQL Attivo: Trigger

Regole ECA (Event-Condition-Action) automatiche.

sqlCREATE TRIGGER LogModifica AFTER UPDATE ON Impiegato 
FOR EACH ROW EXECUTE PROCEDURE LogFunction();

Funzioni e CASE

sqlCASE WHEN Stipendio > 5000 THEN 'High' ELSE 'Standard' END;
COALESCE(Attr, 'N/D'); -- Primo valore non NULL

TCL Avanzato: Savepoints

sqlSAVEPOINT S1; ROLLBACK TO S1;

Esempi Riepilogativi

Interrogazione Dipartimenti: Dipartimenti che spendono più di 100 in stipendi.
SELECT Dipart, SUM(Stipendio) FROM Impiegato GROUP BY Dipart HAVING SUM(Stipendio) > 100;
Interrogazioni con EXCEPT: Dipartimenti senza impiegati di cognome 'Rossi'.
SELECT Nome FROM Dipartimento EXCEPT SELECT Dipart FROM Impiegato WHERE Cognome = 'Rossi';
Esempio Stadi (Subqueries Nidiicate): Nomi degli stadi in cui non gioca nessuna nazionale europea.
sqlSELECT Nome FROM Stadio WHERE Nome NOT IN (
  SELECT NomeStadio FROM Incontro WHERE 
    (Squadra1 IN (SELECT Paese FROM Nazionale WHERE Continente = 'Europa')) OR 
    (Squadra2 IN (SELECT Paese FROM Nazionale WHERE Continente = 'Europa'))
);

E-R Modello Entità-Relazione

Ciclo di vita dei sistemi informativi

La progettazione di una base di dati è solo una componente del ciclo di vita di un sistema informativo, che comprende 6 fasi principali:

# Fase Descrizione
1 Studio di fattibilità Definisce costi delle alternative e priorità di realizzazione.
2 Raccolta e analisi dei requisiti Individua proprietà e funzionalità. Richiede interazione con gli utenti. Produce descrizione informale di dati e operazioni.
3 Progettazione Progettazione dei dati (struttura) e delle applicazioni (programmi). Possono procedere in parallelo.
4 Implementazione Costruzione e popolamento del DB, produzione del codice.
5 Validazione e collaudo Verifica corretto funzionamento in tutte le condizioni operative.
6 Funzionamento Il sistema è operativo. Richiede solo gestione e manutenzione.
⚠ Non è strettamente sequenziale: durante ogni fase si possono rivedere decisioni delle fasi precedenti. Si parla di ciclo di operazioni.

Tre fasi di progettazione

La metodologia standard prevede 3 fasi in cascata, basate sul principio di separare il "cosa" rappresentare dal "come" farlo:

Fase Prodotto Indipendente da
Progettazione concettuale Schema concettuale (diagramma E-R) Qualsiasi DBMS. Rappresenta il "cosa" ad alto livello di astrazione.
Progettazione logica Schema logico (tabelle relazionali) Dettagli fisici. Usa tecniche di normalizzazione. Richiede conoscere il modello logico (es. relazionale) ma non il DBMS specifico.
Progettazione fisica Schema fisico (file, indici) —. Dipende dal DBMS scelto. Specifica parametri fisici di memorizzazione.
💡 Il risultato finale comprende tutti e tre gli schemi: lo schema concettuale (documentazione ad alto livello), lo schema logico (riferimento per query e aggiornamenti) e lo schema fisico (implementazione).

REL Modello Relazionale

Concetti fondamentali

Il modello relazionale, proposto da E.F. Codd nel 1970, è basato su due concetti: relazione (dalla teoria degli insiemi) e tabella (intuitiva). La coesistenza di un concetto formale e uno intuitivo è alla base del suo successo.

Concetto Definizione
Prodotto cartesiano D₁ × D₂ × … × Dₙ — tutte le n-uple possibili dai domini. Grado = numero di componenti.
Relazione matematica Sottoinsieme del prodotto cartesiano. Cardinalità = numero di n-uple.
Relazione nel mod. rel. Come sopra, ma con attributi (non posizionale): ogni colonna ha un nome.
Schema di relazione R(X) — nome R + insieme attributi X. Esempio: Studenti(Matricola, Cognome, Nome, DataNascita)
Istanza di relazione L'insieme effettivo di tuple in un dato momento.
Schema di base di dati Insieme di schemi di relazione: R = {R₁(X₁), R₂(X₂), …, Rₙ(Xₙ)}
Proprietà delle relazioni (come insiemi): non esiste ordinamento fra le tuple; non ci possono essere tuple duplicate; ogni n-upla è internamente ordinata (i valori corrispondono agli attributi).

Attributi, tuple e notazione

Ogni dominio che compare più volte in una relazione deve essere distinto tramite un attributo (nome simbolico). Nella rappresentazione tabellare gli attributi sono le intestazioni delle colonne. 

esempioSquadraDiCasa  | SquadraDiTrasferta | RetiCasa | RetiOspitata
Juventus       | Lazio              | 3        | 1
Lazio          | Milan              | 2        | 0
Juventus       | Roma               | 1        | 2

Se t è una tupla e A un attributo, t[A] indica il valore di t sull'attributo A. Si usa anche per insiemi di attributi: t[SquadraDiCasa, RetiCasa].

💡 Ordine irrilevante: con gli attributi l'ordine delle colonne non è significativo — due tabelle con le stesse righe ma colonne in ordine diverso rappresentano la stessa relazione.

Modello basato su valori

I riferimenti fra dati in relazioni diverse sono rappresentati tramite valori condivisi (tipicamente chiavi), non puntatori. Questo è il principio fondamentale che distingue il modello relazionale dai modelli gerarchico e reticolare.

Vantaggio Spiegazione
Indipendenza fisica La rappresentazione logica (valori) non fa riferimento a quella fisica, che può cambiare nel tempo.
Semplicità di trasferimento I dati sono auto-contenuti; non ci sono puntatori con significato locale al sistema.
Solo dati rilevanti I puntatori sono aspetti realizzativi non significativi per l'applicazione. 
schemaStudenti (Matricola, Cognome, Nome, DataNascita)
Esami    (Studente, Voto, Corso)          -- Studente = matricola → riferimento per valore
Corsi    (Codice, Titolo, Docente)        -- Corso = codice → riferimento per valore

Valori NULL

Il valore NULL rappresenta l'assenza di informazione. Non è un valore del dominio ma un marcatore speciale. Tre significati possibili:

Tipo Significato Esempio
Valore sconosciuto Il valore esiste ma non è disponibile Firenze ha una prefettura ma non ne conosciamo l'indirizzo
Valore inesistente L'attributo non è applicabile per quella tupla Tivoli non è capoluogo → non ha prefettura
Senza informazione Non sappiamo se esiste né il suo valore Provincia di nuova istituzione — status ignoto
⚠ I sistemi relazionali trattano tutti i NULL allo stesso modo (caso "senza informazione"). È necessario moderare la presenza di NULL: su una delle chiavi (chiave primaria) i NULL sono vietati; sulle altre in genere ammessi.

Vincoli di integrità

Un vincolo è un predicato che deve essere vero per tutte le istanze corrette. Le istanze che soddisfano tutti i vincoli sono dette lecite.

Tipo Definizione Esempio
Vincolo di dominio Restrizione sui valori di un singolo attributo Voto >= 18 AND Voto <= 30
Vincolo di tupla Condizione valutabile su ogni tupla indipendentemente NOT(Lode='lode') OR Voto=30
Vincolo intrarelazionale Coinvolge solo tuple/valori di una relazione Unicità della matricola
Vincolo interrelazionale Coinvolge più relazioni Matricola in Esami deve esistere in Studenti

Chiavi

Una chiave è un insieme minimale di attributi che identifica univocamente le tuple.

Concetto Definizione
Superchiave Insieme K tale che non esistono due tuple con gli stessi valori su K.
Chiave Superchiave minimale: nessun suo sottoinsieme proprio è superchiave.
Chiave primaria La chiave scelta come principale. Su di essa i NULL sono vietati. Evidenziata con sottolineatura.
Chiave alternativa Chiave non scelta come primaria. NULL in genere ammessi.
💡 Ogni relazione ha sempre almeno una chiave: nel caso peggiore, l'insieme di tutti gli attributi è superchiave (poiché le tuple sono distinte), da cui si ricava una chiave per riduzione. 
esempioSTUDENTI(Matricola, Cognome, Nome, Nascita, Corso)
-- Chiave 1: {Matricola}
-- Chiave 2: {Cognome, Nome, Nascita}
-- {Matricola, Corso} è superchiave ma NON chiave (non minimale)

Vincoli di integrità referenziale (Foreign Key)

Un vincolo di integrità referenziale fra un insieme di attributi X di R₁ e la relazione R₂ è soddisfatto se i valori su X di ciascuna tupla di R₁ compaiono come valori della chiave primaria di R₂ (oppure sono NULL). 

esempio schemaInfrazioni (Codice, Data, Agente, Articolo, Provincia, Numero)
Agenti      (Matricola, CF, Cognome, Nome)
Auto        (Provincia, Numero, Proprietario, Indirizzo)

-- Vincolo 1: Infrazioni.Agente → Agenti.Matricola
-- Vincolo 2: (Infrazioni.Provincia, Infrazioni.Numero) → (Auto.Provincia, Auto.Numero)
--            L'ORDINE degli attributi è essenziale nelle chiavi composte!
⚠ Attenzione all'ordine: quando la chiave della relazione riferita è composta da più attributi, la corrispondenza va specificata per posizione. Due attributi separati non bastano: serve la combinazione nella stessa tupla.

PL Progettazione Logica

Obiettivo e fasi

L'obiettivo è costruire uno schema logico che descriva correttamente ed efficientemente le informazioni dello schema E-R. Non è una semplice traduzione: lo schema E-R va prima ristrutturato.

Fase Input Output
1. Ristrutturazione E-R Schema E-R + carico applicativo Schema E-R ristrutturato (ottimizzato, senza generalizzazioni né attributi multivalore)
2. Traduzione verso mod. logico Schema E-R ristrutturato + modello logico scelto Schema logico (relazionale) + vincoli di integrità + documentazione

La ristrutturazione si articola in 4 passi in sequenza:

  • Analisi delle ridondanze
  • Eliminazione delle generalizzazioni
  • Partizionamento/accorpamento di entità e associazioni
  • Scelta degli identificatori principali

Analisi delle ridondanze

Una ridondanza è un dato derivabile da altri dati. Va valutato se mantenerla o eliminarla confrontando costo delle operazioni e occupazione di memoria.

Tipo di ridondanza Esempio
Attributo derivabile da attributi stessa entità ImportoLordo = ImportoNetto + IVA
Attributo derivabile da aggregazione su altra entità ImportoTotale di Acquisto = somma dei Prezzi dei Prodotti
Attributo derivabile da conteggio occorrenze NumeroAbitanti di Città = COUNT occorrenze di Residenza
Associazione derivabile da composizione di altre Docenza = Frequenza ∘ Insegnamento
💡 Regola "ottanta-venti": l'80% del carico è generato dal 20% delle operazioni — basta concentrarsi sulle operazioni principali per valutare correttamente le prestazioni.
Strumenti di analisi: tavola dei volumi (numero occorrenze di entità/associazioni), tavola delle operazioni (frequenza e tipo I/B), tavola degli accessi (entità/associazioni visitate per operazione, tipo L/S). Un accesso in Scrittura vale tipicamente il doppio di uno in Lettura.

Eliminazione delle generalizzazioni

Il modello relazionale non supporta direttamente le generalizzazioni. Tre alternative:

Alternativa Come Quando conviene
(1) Accorpamento figlie nel genitore Elimina E₁, E₂; aggiunge i loro attributi a E₀ + attributo Atipo. Possibili NULL. Operazioni che non distinguono tra E₁ ed E₂ (accessi ridotti).
(2) Accorpamento genitore nelle figlie Elimina E₀; copia attributi/identificatore a E₁ ed E₂ per ereditarietà. Solo se generalizzazione è totale. Operazioni su E₁ o E₂ separatamente. No NULL.
(3) Sostituzione con associazioni 1:1 Mantiene E₀, E₁, E₂ separate; aggiunge associazioni RG1 e RG2. Aggiunge vincoli. Generalizzazione non totale, o operazioni che distinguono E₀ da E₁/E₂. Entità con pochi attributi → accesso fisico più efficiente.

Partizionamento/Accorpamento e Scelta identificatori

Partizionamento verticale di entità

Divide un'entità in due collegate da associazione 1:1, separando attributi acceduti da operazioni diverse. Produce entità con pochi attributi → un accesso fisico recupera più tuple.

Accorpamento di entità

Fonde due entità legate da associazione 1:1 in una sola. Conveniente se le operazioni richiedono sempre entrambi i concetti. Possibili NULL se la cardinalità minima è 0.

⚠ Accorpamento su 1:N o N:M genera ridondanze — da evitare o analizzare con la normalizzazione.

Eliminazione attributi multivalore

Un attributo multivalore (es. Telefono di Agenzia) si trasforma in entità separata (Telefono) legata all'entità originale da associazione 1:N.

Scelta degli identificatori principali (chiave primaria)

  • Attributi con possibili NULL → esclusi
  • Preferire identificatori con pochi attributi (indici più piccoli, join più veloci)
  • Preferire identificatori interni agli esterni (le chiavi esterne includono identificatori di altre entità)
  • Preferire identificatori usati più frequentemente dalle operazioni
  • Se nessuno soddisfa: introdurre un attributo artificiale (codice)

ALG Algebra Relazionale

Cos'è l'algebra relazionale

L'algebra relazionale è un linguaggio procedurale basato su un insieme di operatori che prendono relazioni come input e producono relazioni come output. Questo permette di comporre operatori per interrogazioni complesse.

Categoria Operatori
Insiemistici Unione (∪), Intersezione (∩), Differenza (−)
Specifici relazionali Ridenominazione (ρ), Selezione (σ), Proiezione (π)
Join Join naturale (⨝), Prodotto cartesiano, Theta-join, Equi-join, Join esterno

Unione, Intersezione, Differenza

Si applicano solo a relazioni definite sugli stessi attributi (altrimenti il risultato non è una relazione).

Operatore Notazione Risultato
Unione r₁ ∪ r₂ Tuple che appartengono a r₁ oppure r₂ (o entrambe). No duplicati.
Intersezione r₁ ∩ r₂ Tuple che appartengono sia a r₁ sia a r₂.
Differenza r₁ − r₂ Tuple che appartengono a r₁ ma non a r₂.
r₁ ∩ r₂ = r₁ − (r₁ − r₂) — l'intersezione è derivabile dalla differenza.

Ridenominazione (ρ)

Cambia i nomi degli attributi (non i valori). Indispensabile prima di unione/join su relazioni con attributi omonimi che rappresentano concetti diversi. 

notazioneρ Genitore←Padre (Paternità)
-- Rinomina l'attributo "Padre" in "Genitore" nella relazione Paternità

ρ Genitore←Padre (Paternità)  ∪  ρ Genitore←Madre (Maternità)
-- Unisce tutte le coppie genitore-figlio

In generale: ρ B₁B₂…Bₖ ← A₁A₂…Aₖ (r) — si indicano solo gli attributi che cambiano nome.

Selezione (σ)

Produce un sottoinsieme delle tuple (decomposizione orizzontale). Il risultato ha gli stessi attributi dell'operando. 

notazioneσ F (r)     -- F = formula proposizionale

σ Età>30 ∧ Stipendio>4000 (Impiegati)
σ CittàDiNascita = Residenza (Impiegati)   -- confronto tra attributi

La condizione F può usare: = ≠ > < ≥ ≤, confronti tra attributi o tra attributo e costante, connettivi ∧ (and), ∨ (or), ¬ (not).

σ vs WHERE in SQL: la selezione dell'algebra relazionale corrisponde alla clausola WHERE in SQL.

Proiezione (π)

Il theta-join è un operatore derivato: prodotto cartesiano seguito da selezione. Si usa quando le relazioni non hanno attributi in comune ma si vuole correlare per condizione.

NORM La Normalizzazione

Introduzione alla qualità del software

La normalizzazione studia alcune proprietà, dette forme normali, che “certificano” la qualità dello schema di una base di dati relazionale. Quando una relazione non soddisfa una forma normale, presenta ridondanze e si presta a comportamenti poco desiderabili durante le operazioni di aggiornamento.

La teoria della normalizzazione costituisce un utile strumento di verifica in grado di suggerire emendamenti, ma non può sostituire le metodologie di analisi e progettazione di più ampio respiro.

Progetto e verifica: molte considerazioni fatte a livello di schema logico possono essere applicate anche su schemi Entità-Relazione, effettuando analisi della qualità già durante la fase di progettazione concettuale.

Ridondanze e anomalie

Consideriamo una relazione che riunisce concetti fra loro disomogenei. Esempio: impiegati, progetti e partecipazioni.

Impiegato Stipendio Progetto Bilancio Funzione
Rossi 20000 Marte 2000 tecnico
Verdi 35000 Giove 15000 progettista
Verdi 35000 Venere 15000 progettista
Neri 55000 Venere 15000 direttore
Neri 55000 Giove 15000 consulente

Questa relazione presenta tre tipi di anomalie:

  • Anomalia di aggiornamento: se lo stipendio di Verdi cambia, va modificato in tutte le tuple corrispondenti per mantenere la consistenza.
  • Anomalia di cancellazione: se Rossi smette di lavorare al progetto Marte e cancelliamo la tupla, perdiamo anche l'informazione del suo stipendio.
  • Anomalia di inserimento: non possiamo inserire un nuovo impiegato se non è ancora assegnato a un progetto (perché Progetto fa parte della chiave).

Dipendenze Funzionali (FD)

Sia r una relazione su R(X) e Y, Z sottoinsiemi di X. Esiste una dipendenza funzionale Y → Z se, per ogni coppia di tuple t1, t2, se t1[Y] = t2[Y] allora t1[Z] = t2[Z].

  • Esempio: Impiegato → Stipendio (ogni impiegato ha un solo stipendio).
  • Esempio: Progetto → Bilancio (ogni progetto ha un solo bilancio).
  • Banale: Y → Z è banale se Z ⊆ Y.
  • Non banale: Y → Z è non banale se Z ⊈ Y.
Chiave vs FD: se K è una chiave di r, allora per ogni attributo A dello schema vale K → A. Il vincolo di dipendenza funzionale generalizza il vincolo di chiave.

Forma Normale di Boyce e Codd (BCNF)

Una relazione r è in BCNF se per ogni dipendenza funzionale non banale X → A definita su di essa, X è una superchiave per r.

Anomalie e ridondanze si presentano quando esistono dipendenze X → A in cui X non contiene una chiave. In questo caso, i concetti indipendenti sono mescolati in un'unica relazione.

Decomposizione senza perdita (Lossless Join)

Una decomposizione di r su X1, X2 è senza perdita se il join naturale delle proiezioni di r su X1 e X2 restituisce esattamente r (senza tuple spurie).

Condizione sufficiente: r si decompone senza perdita su X1 e X2 se l'intersezione degli attributi X0 = X1 ∩ X2 è chiave per almeno una delle due relazioni (X0 → X1 oppure X0 → X2).

Il join su Sede potrebbe creare tuple spurie se più impiegati lavorano nella stessa sede dove si svolgono più progetti, anche se non partecipano a quei progetti.

Conservazione delle dipendenze

Una decomposizione conserva le dipendenze se tutte le dipendenze funzionali dello schema originario possono essere verificate sulle singole relazioni decomposte, senza dover ricostruire il join.

Se una dipendenza X → A coinvolge attributi che finiscono in relazioni diverse, non è più possibile garantirne il soddisfacimento tramite semplici vincoli di chiave sulle tabelle risultanti.

Qualità: una buona decomposizione deve essere sia senza perdita sia conservativa delle dipendenze.

Terza Forma Normale (3NF)

Talvolta la BCNF non è raggiungibile se vogliamo conservare le dipendenze. Si ricorre quindi alla 3NF, che è meno restrittiva.

Una relazione r è in 3NF se per ogni FD non banale X → A:

  • X contiene una chiave (è superchiave);
  • oppure A è un attributo primo (appartiene ad almeno una chiave).

La 3NF tollera alcune ridondanze ma garantisce che sia sempre possibile ottenere una decomposizione che conservi le dipendenze e sia senza perdita.

Ottenibilità: una qualunque relazione che non soddisfa la 3NF è certamente decomponibile senza perdita e con conservazione delle dipendenze in relazioni in 3NF.

Altre forme normali

Prima Forma Normale (1NF)

Stabilisce che gli attributi delle relazioni devono essere definiti su valori atomici (non insiemi o relazioni).

Seconda Forma Normale (2NF)

Una relazione è in 2NF se è in 1NF e non esistono dipendenze parziali, ovvero dipendenze di attributi non primi da un sottoinsieme proprio della chiave. 

esempioChiave: {Impiegato, Progetto}
FD1: Impiegato -> Stipendio  (Parziale - Viola 2NF)
FD2: Impiegato, Progetto -> Funzione (Completa)

In pratica, se la chiave è composta da un solo attributo, la relazione è automaticamente in 2NF.

Sintesi: 3NF e BCNF sono oggi considerate il miglior compromesso tra semplicità e qualità dei risultati per la maggior parte delle applicazioni.

Modulo 1: Fondamenti di Hardware e Connettività

1.1 Hub (Livello 1 - Fisico)

L'Hub è un dispositivo di rete elementare che opera esclusivamente al Livello 1. Non distingue le porte: quando riceve un segnale, lo rigenera e lo invia a tutti (broadcast).

  • Dominio di Collisione: Tutte le porte appartengono allo stesso dominio. Se due host trasmettono insieme, i segnali si sovrappongono.
  • Efficienza: Molto bassa, cala drasticamente all'aumentare dei dispositivi.

Meccanismi di Accesso: CSMA/CD e CSMA/CA

Protocollo Nome Contesto / Funzionamento
CSMA/CD Collision Detection Ethernet cablata (Hub). Ascolta il mezzo; se rileva collisione, interrompe, invia un jam signal e attende un tempo casuale (backoff).
CSMA/CA Collision Avoidance Wi-Fi (802.11). Non potendo ascoltare e trasmettere insieme, attende un tempo casuale prima di trasmettere e aspetta un ACK.

1.2 Ethernet e Mezzi Fisici

L'interfaccia fisica standard per le LAN è la porta Ethernet (RJ45). La velocità dipende dallo standard e dalla categoria del cavo.

  • FastEthernet (802.3u): Fino a 100 Mbps.
  • GigabitEthernet (802.3ab): Fino a 1 Gbps (standard attuale).
  • Auto-negotiation: I dispositivi si accordano sulla velocità massima supportata da entrambi.
Cavi: Cat5e e Cat6 supportano il Gigabit. La Fibra Ottica invece usa impulsi luminosi per distanze maggiori e velocità fino a 10/100 Gbps.

1.3 Cablaggio Strutturato

Insieme di componenti passivi (cavi, prese, permutatori) installati secondo norme standard (ISO/IEC 11801). Adotta una topologia a stella estesa:

  • CD (Campus Distributor): Centro stella di comprensorio (Main Crossconnect).
  • BD (Building Distributor): Centro stella di edificio.
  • FD (Floor Distributor): Centro stella di piano (Patch Panel in armadio rack).
Limiti di Lunghezza: Il collegamento orizzontale (presa-armadio) è max 90 metri. Il canale totale (inclusi patch cord) è max 100 metri.

Modulo 2: Protocolli di Livello 2 (Data Link)

2.1 ARP (Address Resolution Protocol)

Risolve un IP noto in un MAC sconosciuto nella LAN. Poiché collega L2 e L3, è spesso definito come livello 2.5.

  • ARP Request: Inviata in broadcast ("Chi ha 192.168.1.1?").
  • ARP Reply: Risposta in unicast ("Il mio MAC è AA:BB...").
  • ARP Cache: Tabella temporanea IP ↔ MAC presente in ogni host.
  • RARP: (Obsoleto) Risolve un MAC noto in un IP (sostituito da DHCP).
ARP Spoofing: Vulnerabilità che permette a un attaccante di inviare ARP Reply false per intercettare il traffico (Man-in-the-Middle). IPv6 usa NDP su ICMPv6 invece di ARP.

2.2 Switch e Domini

Lo Switch inoltra i frame in base alla MAC Table (CAM Table).

  • Dominio di Collisione: Ogni porta dello switch è un dominio separato (Full-duplex).
  • Dominio di Broadcast: Di default, tutte le porte appartengono allo stesso dominio.

2.3 STP (Spanning Tree Protocol)

Evita i loop di livello 2 in caso di collegamenti ridondanti, disattivando logicamente alcuni link.

  1. Elezione Root Bridge: Lo switch con Bridge ID più basso diventa il centro stella logico.
  2. Calcolo Path Cost: Ogni switch sceglie la porta migliore (Root Port) verso il root.
  3. Blocking: Le porte ridondanti vengono bloccate; si riattivano solo in caso di guasto.

2.4 VLAN e Trunking

Le VLAN segmentano logicamente lo switch fisico in più domini di broadcast.

  • Porta Access: Appartiene a una sola VLAN (es. per i PC).
  • Porta Trunk: Trasporta più VLAN (tag 802.1Q) tra switch o verso il router.
  • Router-on-a-stick: Uso di una singola porta fisica del router divisa in sub-interfacce logiche per il routing inter-VLAN.

Indirizzamento e Subnetting

Un indirizzo IPv4 è un numero di 32 bit diviso in 4 ottetti. Si compone di una Parte Network (identifica la rete) e una Parte Host (identifica il dispositivo).

Subnet Mask e Operazione AND

La subnet mask indica dove finisce la rete e dove iniziano gli host. Il router esegue un AND bit a bit tra l'IP e la maschera per ricavare l'indirizzo di rete. 

Esempio ANDIP:      192.168.1.132 -> 11000000.10101000.00000001.10000100
MASK:    255.255.255.0 -> 11111111.11111111.11111111.00000000
NETWORK: 192.168.1.0   -> 11000000.10101000.00000001.00000000

Classi IP e CIDR

Originariamente gli IP erano divisi in classi fisse (A, B, C), ma questo causava sprechi. Nel 1993 è stato introdotto il CIDR (Classless Inter-Domain Routing), che usa la notazione /N per indicare il numero di bit a 1 nella maschera.

FLSM vs VLSM

  • FLSM (Fixed Length): Tutte le sottoreti hanno la stessa dimensione. Semplice ma inefficiente.
  • VLSM (Variable Length): Ogni sottorete è dimensionata sul numero reale di host necessari. Ottimizza lo spazio.
Regola VLSM: Ordinare sempre le richieste di host in modo decrescente (dalla più grande alla più piccola) prima di assegnare gli indirizzi.

Modulo 3: Servizi di Rete e Indirizzamento

3.1 DHCP (Dynamic Host Configuration Protocol)

Assegna automaticamente IP, mask, gateway e DNS agli host. Usa UDP (porte 67 e 68).

  • Processo DORA: Discover (broadcast), Offer, Request, Acknowledgment (ACK).
  • Lease Time: Tempo di "noleggio" dell'indirizzo.

5.3 AAA e RADIUS

Framework per la gestione centralizzata degli accessi:

  • Authentication: Chi sei? (Username/Password).
  • Authorization: Cosa puoi fare? (Privilegi, comandi).
  • Accounting: Cosa hai fatto? (Log delle attività).
  • RADIUS: Protocollo standard (UDP 1812/1813) per l'autenticazione remota.

5.4 VPN (Virtual Private Network)

Estende una rete privata su infrastruttura pubblica tramite Tunneling e cifratura.

  • Site-to-Site: Collega due sedi aziendali in modo trasparente.
  • Remote Access: Un singolo utente si collega alla sede (es. Smart Working).
  • IPsec Phases: Fase 1 (IKE) stabilisce il canale sicuro; Fase 2 (IPsec) cifra i dati reali.

Modulo 6: Reti Wireless (WLAN)

Infrastruttura che integra il dominio radio (802.11) con quello cablato (802.3).

6.1 Meccanismo CSMA/CA

A differenza di Ethernet, il Wi-Fi non può rilevare collisioni (CD). Usa la Collision Avoidance:

  • Ascolto: Se il canale è libero, attende un tempo casuale (backoff) prima di trasmettere.
  • ACK: Il destinatario deve confermare ogni frame. Se l'ACK non arriva, la stazione presume una collisione e ritenta.

6.2 Organizzazione del Servizio

  • BSS (Basic Service Set): Un Access Point e i suoi client associati. Identificato dal BSSID (MAC dell'AP).
  • SSID (Service Set Identifier): Nome logico della rete (es. "WiFi-Scuola"). Annunciato dai Beacon.
  • ESS (Extended Service Set): Più BSS collegati allo stesso switch, permettono il Roaming (spostamento tra AP senza caduta).

6.3 Architettura Controller-Based

In reti grandi, gli AP non sono autonomi ma gestiti da un WLC (Wireless LAN Controller).

  • Lightweight AP (LAP): AP "leggeri" che delegano la gestione al controller.
  • Split-MAC Architecture: L'AP gestisce le funzioni radio real-time; il WLC gestisce config, sicurezza e associazioni.
  • CAPWAP: Protocollo di tunneling tra AP e WLC.
  • DHCP Option 43: Permette agli AP di scoprire automaticamente l'IP del WLC.
FlexConnect: Modalità che permette agli AP di continuare a funzionare localmente anche se perdono la connessione col WLC.

Tecnologie Emergenti e Approfondimenti

Cloud Computing

Erogazione di servizi (storage, server, DB) tramite Internet con modello pay-as-you-go.

  • Modelli di Deployment: Public (risorse condivise), Private (dedicato), Hybrid (mix dei due).
  • Multi-tenancy: Più utenti (tenant) condividono le stesse risorse fisiche, isolati via software.
Modello Nome Esempio / Funzione
IaaS Infrastructure as a Service VM, reti, storage. Utente gestisce OS e App. (es. AWS, Azure).
PaaS Platform as a Service Ambienti di sviluppo e DB. Utente carica solo il codice.

Crittografia

Tecnica per rendere i messaggi incomprensibili ai non autorizzati.

  • Simmetrica: Stessa chiave per cifrare e decifrare (es. AES). Veloce ma difficile distribuire le chiavi.
  • Asimmetrica: Coppia di chiavi: Pubblica (diffusa) e Privata (segreta). Un messaggio cifrato con la pubblica si apre solo con la privata (es. RSA).
  • Metodo Ibrido: Si usa la crittografia asimmetrica solo per scambiarsi una chiave simmetrica (più veloce per i dati reali).
Hash (MD5, SHA-2): Funzione unidirezionale che genera un'impronta fissa di un dato. Usata per verificare l'integrità (se il file cambia, l'hash cambia).

Firma Digitale: Documento cifrato con la chiave privata del mittente. Garantisce autenticità e non ripudio.
Certificato Digitale: Documento rilasciato da una CA (Certification Authority) che associa una chiave pubblica a un'identità certa.
